Tab warnings: Expo Router registers directory dynamic routes with their full segment, e.g. "appointment-request/[serviceId]" and "business/[id]". The name prop must match exactly. Fixed by using the full segment names.

Session error: After amplifySignOut() in logout, Amplify clears both in-memory token cache AND SecureStore (via the async React Native adapter). When amplifySignIn() runs next, it writes new tokens to SecureStore asynchronously. fetchAuthSession() immediately after signIn reads from the just-cleared in-memory cache (finds nothing), then tries storage (write hasn't completed yet), returns null → our error is thrown. The forceRefresh:true retry path also fails for the same reason (can't exchange refresh token if it's not in storage yet). Fix: retry fetchAuthSession() after a 500ms delay to let the async SecureStore write settle. Removed forceRefresh (makes a needless network round-trip).
